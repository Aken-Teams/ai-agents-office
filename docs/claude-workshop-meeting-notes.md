# Claude WorkShop 協作平台 — 需求會議紀錄

> **需求編號**: REQ-2026-017
> **會議日期**: 2026-03-30
> **會議主題**: Claude WorkShop 協作平台需求確認
> **階段**: 需求確認 → MVP 架構確認
> **估計 SP**: 35 SP

---

## 一、背景說明

公司已有部分團隊在日常開發中導入 Claude 等 AI 工具，顯著提升工作效率。但隨著使用範圍擴大，暴露出海外廠區不可用、資料安全不可控、開發成果散落等問題。本次會議釐清現況痛點，確認 Claude WorkShop 平台的建設方向與期待效益。

---

## 二、需求說明

1. **地端化 Claude 部署** — 在公司內網架設 Claude 服務，AI 運算與資料處理全程不離開企業環境，確保程式碼與機敏資料零外洩
2. **海外廠區統一存取** — 海外據點（大陸廠、東南亞廠等）透過 VPN / 內網即可使用與總部相同的 Claude 服務，消除區域限制造成的工具斷層
3. **統一 AI 開發入口** — 提供公司級的 Web 平台，取代各團隊自行尋找雲端工具的現況，統一工具版本、Prompt 範本與使用規範
4. **對話式 AI 助手** — 透過聊天介面與 AI 互動，支援多輪對話、檔案上傳分析、程式碼輔助撰寫、文件自動生成等
5. **多 Agent 智慧協作** — 系統自動分析需求，拆解為研究、規劃、生成等步驟，派遣多個專業 Agent 串行或並行完成複雜任務
6. **文件自動生成** — AI 輔助產出 PPT、Word、Excel、PDF、互動網頁簡報等商業文件，內建多種專業範本與風格
7. **內部 App 工具庫** — 開發者用 AI 產出的工具、腳本、小型應用可上架至內部平台，其他同仁可搜尋、預覽、一鍵部署使用
8. **新手引導與範本系統** — 預設常用工作流程範本與 Prompt 範例，降低初階開發者的上手門檻，減少資深工程師重複教學時間
9. **用量監控與計費** — Token 使用量追蹤、部門級計費報表、個人配額管理，用量透明可稽核
10. **權限與角色管理** — 管理員 / 一般使用者分級、部門分群、功能存取控制、敏感操作審計日誌
11. **檔案版本控制** — 所有 AI 生成的文件自動版本化，支援歷史版本回溯與下載，人員異動不影響成果留存
12. **管理後台** — 系統總覽、使用者管理、技能管理、Token 帳本、安全監控、系統設定等一站式管理介面

## 三、現況痛點

### 痛點 1：海外廠區無法使用 Claude

- 大陸廠、東南亞廠等海外據點因區域與網路限制，**完全無法存取 Claude 服務**
- 海外工程師被迫使用其他替代工具甚至純手動開發，**效率遠低於總部團隊**
- 同一個專案跨廠區協作時，AI 工具不一致導致**產出品質與格式落差大**
- 海外同仁無法參與公司推動的 AI 輔助開發流程，形成**數位落差**

### 痛點 2：AI 工具各自為政，資料上雲失控

- 各團隊、甚至各人**自行尋找雲端 AI 工具**（ChatGPT、Copilot、各種 SaaS 平台）
- 開發過程中程式碼、內部文件、客戶資料**可能被上傳至第三方雲端**
- 公司**無從得知**哪些敏感資料被送出去、送去哪裡、對方怎麼使用
- 沒有統一的使用紀錄與稽核機制，**資安合規形同虛設**
- 工具版本與能力參差不齊，**團隊間協作成本高**

### 痛點 3：開發成果散落，無法留存與複用

- 工程師用 AI 輔助開發的小工具、自動化腳本、內部 App，**散落在個人電腦裡**
- 沒有統一平台可以保存、版本管理、分享這些成果
- 相同需求被不同人**重複造輪子**，浪費開發資源
- 人員離職或調動時，**知識與成果直接流失**，新人得從頭摸索
- 好的實踐經驗無法沉澱為組織資產

### 痛點 4：小白開發者壓力大，上手門檻高

- 初階工程師或非工程背景人員想用 AI 工具，但**環境設定、API 串接門檻高**
- 從安裝 CLI → 設定 API Key → 寫 Prompt → 整合到工作流，**每一步都卡關**
- 缺乏標準化範本與引導流程，新手**不知道從哪裡開始**
- 資深工程師被迫花時間手把手教學，**拖慢整體開發進度**
- 部署產出物也是門檻：新手做完東西不知道怎麼讓別人用

## 四、預期效益

### 效益 1：資料安全可控，零外洩

- 所有 AI 運算與對話資料**留在公司地端**，不經過任何第三方雲端
- 統一管控資料流向，敏感資訊（程式碼、客戶資料、內部文件）**不出廠**
- 完整的操作日誌與稽核軌跡，**滿足資安合規要求**

### 效益 2：海外廠區同步賦能

- 海外據點透過 VPN 即可使用**與總部完全相同的 Claude 服務**
- 消除跨廠區的工具落差，**統一開發體驗與產出品質**
- 全球團隊在同一平台上協作，降低溝通與整合成本

### 效益 3：AI 工具統一化，告別各自為政

- 提供**公司級的 AI 開發入口**，不再需要各自找工具
- 管理者掌握全局：誰在用、用多少、做了什麼，**用量透明可追蹤**
- 統一的 Prompt 範本與最佳實踐，**團隊產出一致性提升**

### 效益 4：開發成果沉澱，知識不流失

- 開發者產出的工具與 App 可以**上架到內部工具庫**
- 其他同仁可瀏覽、搜尋、**一鍵部署**，不需重複開發
- 形成公司內部的 **App Store 生態**，持續累積組織知識資產
- 人員異動不影響成果留存，**新人可直接複用前人的工作**

### 效益 5：降低門檻，加速全員 AI 化

- 對話式介面 + 預設範本，**小白開發者 30 分鐘內上手**
- 不需自己搞 CLI 安裝、API 設定，平台全部包辦
- 一鍵部署機制讓新手也能**把做好的東西分享給別人用**
- 資深工程師從重複教學中解放，**專注在高價值工作上**
