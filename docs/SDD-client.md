# AI Agents Office — 系統設計文件（客戶版）

> 版本：1.0｜日期：2026-03-25

---

## 1. 文件目的

本文件說明 AI Agents Office 的系統架構設計，幫助客戶了解系統的整體運作方式、部署模型、安全設計與擴展能力。

---

## 2. 系統架構總覽

### 2.1 部署架構

AI Agents Office 採用**本機部署**架構，所有元件運行在客戶自有環境中：

```
┌──────────────────────────────────────────────┐
│                  客戶環境                      │
│                                              │
│  ┌──────────┐    ┌───────────┐               │
│  │ 瀏覽器    │◄──▶│ Web 伺服器 │               │
│  │ (使用者)  │    │ (前端+後端)│               │
│  └──────────┘    └─────┬─────┘               │
│                        │                     │
│           ┌────────────┼────────────┐        │
│           │            │            │        │
│      ┌────▼───┐  ┌────▼────┐ ┌────▼─────┐  │
│      │ 資料庫  │  │ AI 引擎  │ │ 檔案儲存  │  │
│      │        │  │ (本機)   │ │ (本機)    │  │
│      └────────┘  └─────────┘ └──────────┘  │
│                                              │
└──────────────────────────────────────────────┘
```

**資料不離開客戶環境**——所有運算、儲存、AI 推理皆在本機完成。

### 2.2 系統組成

| 元件 | 功能 | 說明 |
|------|------|------|
| Web 前端 | 使用者介面 | 對話界面、檔案管理、管理後台 |
| API 後端 | 業務邏輯 | 請求處理、安全檢查、Agent 協調 |
| AI 引擎 | 智慧生成 | 自然語言理解、內容生成 |
| 資料庫 | 資料持久化 | 使用者帳號、對話紀錄、檔案記錄 |
| 檔案儲存 | 文件管理 | 生成文件、上傳檔案、版本備份 |

## 3. 核心設計

### 3.1 多 Agent 智慧協調

系統採用**多 Agent 協調架構**，將複雜的文件需求自動拆解為多個專業步驟：

```
使用者需求
    │
    ▼
┌──────────┐
│ 路由 Agent │  分析需求，決定需要哪些步驟
└────┬─────┘
     │ 派遣任務
     ▼
┌──────────────────────────────────┐
│          Worker Agents           │
│                                  │
│  ┌────────┐  ┌────────┐         │
│  │ 研究   │  │ 規劃   │  串行    │
│  │ Agent  │─▶│ Agent  │  執行    │
│  └────────┘  └────┬───┘         │
│                   │              │
│  ┌────────┐  ┌────▼───┐         │
│  │ Word   │  │ PPT    │  並行    │
│  │ Agent  │  │ Agent  │  執行    │
│  └────────┘  └────────┘         │
└──────────────────────────────────┘
     │
     ▼ 結果回報
┌──────────┐
│ 路由 Agent │  檢查結果，決定是否需要下一輪
└──────────┘
```

**設計特點：**

| 特性 | 說明 |
|------|------|
| 自動任務拆解 | 路由 Agent 判斷最佳執行策略 |
| 串行與並行 | 有依賴關係的步驟按順序執行，無依賴的可同時進行 |
| 多輪修正 | 最多 3 輪迭代，確保產出品質 |
| 即時透明 | 每個步驟的進度即時回報前端 |

### 3.2 可用的專業 Agent

| Agent | 專長 | 產出 |
|-------|------|------|
| 研究 Agent | 網路搜尋、資料蒐集 | 研究報告文字 |
| 規劃 Agent | 大綱規劃、策略建議 | 結構化大綱 |
| 審查 Agent | 品質檢查、一致性審核 | 修改建議 |
| 資料分析 Agent | CSV/Excel 數據分析 | 分析結果 |
| 跨文件分析 Agent | 多文件比對與摘要 | 綜合分析 |
| PowerPoint Agent | 簡報生成 | .pptx 檔案 |
| Word Agent | 文件生成 | .docx 檔案 |
| Excel Agent | 試算表生成 | .xlsx 檔案 |
| PDF Agent | PDF 生成 | .pdf 檔案 |
| Web 簡報 Agent | 互動式簡報生成 | .html 檔案 |

### 3.3 執行模式

系統支援兩種執行模式，自動判斷使用哪一種：

| 模式 | 觸發條件 | 說明 |
|------|----------|------|
| 智慧協調模式 | 使用者未指定文件類型，或需求較複雜 | 路由 Agent 分析後自動拆解並派遣 |
| 直接生成模式 | 使用者明確指定文件類型 | 直接派遣對應 Agent 執行 |

## 4. 即時串流設計

### 4.1 即時回饋機制

系統採用 **Server-Sent Events (SSE)** 技術，在文件生成過程中即時回報進度：

```
使用者送出需求
    │
    ▼
┌─── 即時串流通道 ────────────────────────────┐
│                                             │
│  ⏱ AI 正在分析您的需求...                    │
│  📋 任務規劃完成：研究 → 規劃 → 生成簡報      │
│  🔍 研究 Agent 開始搜尋資料...               │
│  📊 研究 Agent 完成，找到 5 篇相關報告         │
│  📝 規劃 Agent 正在擬定大綱...               │
│  🔧 PPT Agent 正在生成簡報...                │
│  📁 檔案已生成：提案簡報.pptx (2.4 MB)       │
│  ✅ 完成！共使用 12,450 tokens               │
│                                             │
└─────────────────────────────────────────────┘
```

### 4.2 回報內容

| 類別 | 說明 |
|------|------|
| AI 文字輸出 | 助手的文字回覆，即時逐字顯示 |
| 工具活動 | AI 正在執行的操作（搜尋、寫檔、分析等） |
| 任務進度 | 多 Agent 模式下，各任務的啟動/完成狀態 |
| 檔案通知 | 新檔案生成時立即通知並可預覽 |
| 用量統計 | Token 使用量即時更新 |

## 5. 安全架構

### 5.1 多層防禦模型

系統採用 **5 層縱深防禦** 確保安全：

```
第 1 層：輸入檢測
  ──────────────────────
  所有使用者輸入經過 7 種安全偵測器分析
  自動攔截惡意指令、注入攻擊、路徑穿越等

第 2 層：路徑驗證
  ──────────────────────
  所有檔案操作限制在使用者專屬目錄
  阻擋任何嘗試存取其他區域的行為

第 3 層：AI 工具限制
  ──────────────────────
  AI Agent 只能使用白名單內的工具
  高危操作（刪除、系統指令等）被完全封鎖

第 4 層：行為規則
  ──────────────────────
  每個 Agent 啟動時注入安全規則
  從源頭約束 AI 的行為範圍

第 5 層：檔案隔離
  ──────────────────────
  每位使用者獨立工作空間
  每次對話獨立沙箱環境
  使用者之間完全隔離
```

### 5.2 輸入安全檢測

系統內建 7 種偵測器，自動分析每筆使用者輸入：

| 偵測項目 | 說明 |
|----------|------|
| 指令注入 | 偵測嘗試控制 AI 行為的惡意指令 |
| 語意攻擊 | 辨識自然語言形式的繞過嘗試（中英文） |
| 編碼混淆 | 偵測 Unicode 零寬字元、方向覆寫等隱藏手段 |
| 編碼載荷 | 辨識 Base64 等編碼形式的隱藏指令 |
| 路徑穿越 | 阻擋企圖存取系統目錄的路徑操作 |
| 公式注入 | 防範 Excel 公式攻擊 |
| 升權嘗試 | 偵測多輪對話中的漸進式繞過 |

檢測結果分為三級處理：

| 等級 | 分數 | 處理 |
|------|------|------|
| 安全 | < 30 | 正常通過 |
| 警告 | 30-59 | 允許通過但記錄事件 |
| 封鎖 | ≥ 60 | 拒絕請求並記錄 |

所有安全事件記錄在審計日誌中，管理員可在後台查閱。

### 5.3 檔案隔離架構

```
工作空間/
└── 使用者 A/
│   ├── 上傳檔案/          ← A 的上傳
│   ├── 對話 1/           ← 每次對話獨立空間
│   │   ├── 報告.docx
│   │   └── 數據.xlsx
│   └── 對話 2/
│       └── 簡報.pptx
│
└── 使用者 B/              ← 完全隔離，互不可見
    ├── 上傳檔案/
    └── 對話 1/
        └── 提案.pptx
```

### 5.4 認證與權限

| 功能 | 說明 |
|------|------|
| 帳號密碼登入 | 密碼經加密儲存，不可逆 |
| Google SSO | 支援企業 Google 帳號單一登入 |
| 登入保護 | 5 次失敗 → 15 分鐘冷卻鎖定 |
| 請求限速 | 每分鐘 30 次，防止濫用 |
| 角色控管 | 一般使用者 / 管理員 兩級權限 |
| Token 效期 | 登入憑證 7 天後自動過期 |

## 6. 資料管理

### 6.1 資料分類

| 資料類別 | 儲存位置 | 保留策略 |
|----------|----------|----------|
| 使用者帳號 | 資料庫 | 長期保留 |
| 對話紀錄 | 資料庫 | 使用者可刪除 |
| 生成的文件 | 本機磁碟 | 使用者可管理 |
| 上傳的檔案 | 本機磁碟 | 使用者可刪除 |
| Token 用量 | 資料庫 | 長期保留（計費用） |
| 安全事件 | 資料庫 | 長期保留（稽核用） |

### 6.2 檔案版本管理

系統自動追蹤文件版本：

```
第 1 次生成 → 報告.docx (v1)
第 2 次修改 → 報告.docx (v2)，v1 自動備份
第 3 次修改 → 報告.docx (v3)，v1/v2 皆保留

使用者可隨時下載任何歷史版本
```

### 6.3 上傳檔案處理

```
使用者上傳檔案
    │
    ├─ 大小/類型檢查
    ├─ 安全掃描
    │   ├─ 安全 → 可正常使用
    │   ├─ 可疑 → 允許但標示警告
    │   └─ 危險 → 拒絕上傳
    │
    └─ 儲存至使用者專屬空間
```

## 8. 效能與擴展性

### 8.1 效能設計

| 面向 | 設計 |
|------|------|
| 並行處理 | 多個 Agent 可同時執行，縮短複雜任務時間 |
| 串流回應 | 不等全部完成，即時回傳進度 |
| 連線池 | 資料庫連線重用，避免頻繁建立 |
| 前端快取 | 翻譯資源按需載入，減少初始載入時間 |

### 8.2 超時保護

| 項目 | 上限 |
|------|------|
| 單一 Agent 任務 | 1.5 ~ 10 分鐘（依任務類型） |
| 整體協調流程 | 15 分鐘 |
| 協調輪次 | 最多 3 輪 |

超時時若已有部分產出，系統會保留已完成的部分，不會完全白費。

### 8.3 資源配額

| 項目 | 預設值 | 可調整 |
|------|--------|--------|
| 使用者儲存空間 | 2 GB | 管理員可調 |
| 上傳檔案大小 | 500 MB | 管理員可調 |
| 每分鐘請求數 | 30 次 | 管理員可調 |
| 使用費用上限 | $50 / 使用者 | 管理員可調 |

## 9. 部署規格

### 9.1 環境需求

| 項目 | 最低需求 | 建議配置 |
|------|----------|----------|
| 作業系統 | Windows 10+ / macOS / Linux | — |
| CPU | 4 核心 | 8 核心以上 |
| 記憶體 | 8 GB | 16 GB 以上 |
| 磁碟空間 | 20 GB | 100 GB SSD |
| 網路 | 需要網路連線 | 穩定寬頻 |

### 9.2 軟體相依

| 軟體 | 用途 |
|------|------|
| Node.js 18+ | 應用程式執行環境 |
| MySQL 8.x | 資料儲存 |
| Claude CLI | AI 推理引擎 |
| 現代瀏覽器 | 使用者端存取 (Chrome/Edge/Firefox/Safari) |

### 9.3 網路需求

| 連線方向 | 用途 | 必要性 |
|----------|------|--------|
| 本機 ↔ AI API | AI 推理請求 | 必要 |
| 本機 ↔ 網際網路 | 研究 Agent 搜尋資料 | 選用（研究功能需要） |
| 瀏覽器 ↔ 本機 | 使用者操作 | 必要 |

### 9.4 安裝流程概述

```
1. 安裝基礎環境（Node.js、MySQL、Claude CLI）
2. 下載並設定應用程式
3. 初始化資料庫
4. 設定管理員帳號
5. 啟動服務
6. 透過瀏覽器存取
```

預計安裝時間：30 分鐘（含環境設定）。
